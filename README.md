# 專業電商數據驅動內容行銷 SOP

本文件詳細描述了專業行銷公司如何整合多元數據指標，為電商客戶產出具備高轉換潛力的影音與文章內容。

## 1. 數據架構圖
我們整合以下五大維度的數據：
- **流量數據 (GA4):** 用戶來源、停留時間、跳出率。
- **搜尋數據 (GSC):** 點擊次數、曝光次數、進站關鍵字、排名。
- **銷售數據 (ERP/CMS):** 熱銷品項、平均客單價 (AOV)、新舊客比例。
- **社群輿情 (Scraper):** 熱門討論主題、用戶痛點、競爭對手評價。
- **廣告數據 (Meta/Google Ads):** 點擊成本 (CPC)、廣告投資報酬率 (ROAS)、開場留存率。

## 2. 標準作業流程 (Workflow)

### 第一階段：診斷與策略 (Diagnosis & Strategy)
*   **目標：** 確定內容的「方向」與「成功指標」。
*   **動作：**
    *   分析 GSC 中「高曝光、低點擊」的關鍵字，定為文章優化標的。
    *   檢視 GA4 中不同來源的轉換路徑，決定影片分發的主平台。

### 第二階段：痛點挖掘 (Pain Point Extraction)
*   **目標：** 找出能引發共鳴的「內容鉤子 (Hook)」。
*   **動作：**
    *   使用爬蟲抓取 Dcard、FB、PTT 討論，分析用戶對特定品類的「解決方案需求」。
    *   將銷售數據中「常被加購」的組合做為內容組合推薦。

### 第三階段：內容產出 (Production)
*   **目標：** 高品質影音與專業文案。
*   **動作：**
    *   **短影音：** 針對 Meta Ads 數據回測，設計前 3 秒強留存畫面。
    *   **SEO 文章：** 根據 GSC 關鍵字雲，產出具備權威性 (E-E-A-T) 的長文。

### 第四階段：測試與分發 (Test & Distribution)
*   **目標：** 最小化成本，最大化效益。
*   **動作：**
    *   先以 1/10 預算進行 A/B Testing。
    *   觀察即時 GSC 排名變化，動態調整文章標題。

### 第五階段：回饋與優化 (Iterative Optimization)
*   **目標：** 建立數據循環。
*   **動作：**
    *   將最終轉換數據導回 AI 模型，優化下一次的企劃模板。
